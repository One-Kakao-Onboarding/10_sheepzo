# PRD: AI 기반 IP 캐릭터–배우 캐스팅 추천 시스템 (MVP)

## 1. 제품 개요

### 1.1 문제 정의

웹소설·웹툰 IP를 드라마/영화로 영상화하는 과정에서

캐스팅 담당자와 엔터 기획자는 **‘이 캐릭터에 가장 적합한 배우를 찾는 문제’**에 지속적으로 어려움을 겪고 있다.

- 캐릭터 해석은 주관적이며
- 기존 유명 배우 풀(pool)에 의존하거나
- 추천·경험·감에 기반한 의사결정이 반복된다

그 결과,

- 원작 팬들의 캐스팅 불만
- “캐스팅 실패작”이라는 낙인
- 캐릭터에 잘 맞지만 덜 알려진 배우의 기회 박탈이라는 구조적 문제가 발생한다.

본 제품은 **AI를 활용해 캐릭터 분석과 배우 데이터를 정량적으로 연결하여,
캐스팅 의사결정을 보조하는 AI 추천 도구**를 제공한다.

---

## 2. 목표 (Goals)

### 2.1 MVP 목표

- 특정 캐릭터 설명 텍스트와 배우 텍스트 데이터셋을 입력하면
- **해당 캐릭터에 적합한 배우 후보 TOP N명을 점수, 추천사유와 함께 추천**한다

### 2.2 비목표 (Non-goals)

- 캐스팅 자동 결정 ❌
- 계약, 출연료 협상 ❌
- 실제 흥행 성과 예측 ❌
→ **의사결정 보조용 추천 도구에 집중**

---

## 3. 사용자 정의 (Persona)

### 3.1 핵심 사용자

- 웹소설/웹툰 IP 영상화 프로젝트의
    - 캐스팅 담당자
    - 엔터테인먼트 기획자
    - 제작 PD

### 3.2 사용자 니즈

- 캐릭터에 대한 해석을 정리해주길 원함
- “왜 이 배우가 어울리는지” 설명이 필요함
- 기존 인맥·풀에 없는 배우도 보고 싶음

---

## 4. Pain Points

### 4.1 메인 Pain Point

- 캐릭터에 어울리는 배우 후보를 **체계적으로 찾기 어렵다**
- 캐스팅 실패에 대한 리스크 부담이 크다

### 4.2 부수 Pain Point

- 원작 팬들의 캐스팅 불만
- 신인/비주류 배우의 기회 박탈
- 캐스팅 논의 시 객관적 근거 부족

---

## 5. 경쟁 환경 및 기존 대안

| 기존 방식 | 한계 |
| --- | --- |
| 내부 배우 풀 활용 | 선택지 제한 |
| 추천 기반 캐스팅 | 주관성·편향 |
| 감독/캐스팅 디렉터 감 | 재현 불가, 설명 어려움 |

→ **정량적 분석 + 설명 가능한 추천**이 부재

## 6. 핵심 사용자 플로우 (MVP)

### Step 1. 캐릭터 정보 업로드

- 사용자는 소설/웹툰 내 **특정 캐릭터에 대한 설명을 담은 텍스트**를 업로드한다
- 입력 예시:
    - 캐릭터 소개 문단
    - 성격, 외모 묘사
    - 서사/배경 설명

### Step 2. 배우 데이터셋 업로드

- 사용자는 배우별 텍스트 데이터셋을 업로드한다
- MVP 기준:
    - 배우 1명당 텍스트 파일을 업로드 한다
    - 나무위키 텍스트 복사본 사용
        - 추후 기능 고도화 시 나무위키 외 타 정보(ex. 팬들의 특정 배우에 대한 반응) 등이 추가 업로드 대상이 될 수 있음

### Step 3. AI 분석 및 추천

- AI는
    - 캐릭터 분석 결과
    - 배우 데이터셋
    을 비교 분석하여

→ **캐릭터에 가장 어울리는 배우 TOP N명 + 점수 + 추천 사유**를 출력한다

## 7. 핵심 기능 상세

### 7.1 캐릭터 분석 기능

### Input

- 캐릭터 설명 텍스트 (Plain Text)

### AI 처리

- LLM 기반 텍스트 분석
- 아래 5가지 요소를 구조화하여 추출한다. 없는 정보는 만들지 말고 없음으로 표시해야한다.
- 각 요소에 대해 llm 은 서술형으로 작성해 저장해야한다
    1. 외적 이미지 (나이대, 분위기, 성별, 목소리, 키/덩치 등) 
    2. 성격 스펙트럼
    3. 서사적 역할 (주인공, 빌런, 성장형, 직업, 사회적 위치 등)
    4. 감정 스펙트럼
        - 감정 폭이 큰가, 미세한가
        - 눈빛/침묵 연기가 중요한 캐릭터인가
        - 울음, 분노, 절망 같은 극단 감정 빈도

### Output (예시)

- 키워드가 아니라 서술형으로 적어도 된다

```json
{
  "personality": ["냉철함", "이중성"],
  "age_range": "30대 초중반",
  "tone": ["어두움", "긴장감"],
  "archetype": ["반영웅", "천재형"]
}

```

### 7.2 배우 데이터 정규화 기능

### Input

- 배우별 텍스트 데이터셋 (나무위키 기반)

### AI 처리

- 동일한 기준으로 정보 추출
- 아래 요소를 구조화하여 추출한다. 없는 정보는 만들지 말고 없음으로 표시해야한다.
1. 외적 이미지 (나이대, 분위기, 성별, 목소리, 키/덩치 등) 
    - 기본 배우 프로필
        - 이름
        - 나이대
        - 성별
    - 비주얼
        - 키, 덩치
        - 목소리
        - 인상
2. 성격 스펙트럼
3. 서사적 역할(기존 이미지)
    - 주로 맡았던 배역
    - 과거 배역 설명 텍스트
    - 업로드 데이터에서 반복되는 이미지
    - 주요 감정연기 경험 패턴 (ex. 분노, 절망, 죄책감, 사랑 등)
4. 감정 스펙트럼 (연기톤 스펙트럼)
    - 차가움 ↔ 따뜻함
    - 능동 ↔ 수동
    - 감정 폭: 미세 ↔ 강함
    - 외향 ↔ 내향
    - 코믹
- 

### Output (예시)

- 키워드가 아니라 서술형으로 적어도 된다

```json
{
"actor_name":"OOO",
"image_keywords":["차가움","지적"],
"role_history":["천재","빌런"],
"public_perception":["연기력 호평"],
"age_range":"30대"
}

```

### 7.3 캐릭터–배우 매칭 로직

### 방식 (MVP 기준)

- 캐릭터 특징 벡터
- 배우 특징 벡터
    
    를 비교하여 **유사도 점수 산출**
    

### 추천 기준 선택 (예시 가중치)

유저가 추천에 있어서의 기준 우선순위를 선택할 수 있는 UI 가 필요하다. 

아래는 기준에 대한 예시이다.

- 성격/이미지 유사도: 40%
- 역할 경험 적합도: 30%
- 나이대/비주얼 조건: 20%
- 대중 인식 리스크: 10%

### AI 추천

- 유저가 선택한 우선순위를 참고하여 AI 가 캐릭터에 적합한 배우 리스트 n명을 점수 및 추천 사유와 함께 출력한다.

### Output 예시

```json
[
{
"actor_name":"OOO",
"score":87,
"reason":"냉철한 이미지와 반영웅 역할 경험이 캐릭터와 매우 유사"
},
{
"actor_name":"XXX",
"score":79,
"reason":"유사한 서사형 캐릭터 경험은 적지만 이미지 적합도가 높음"
}
]

```

## 10. 향후 확장 기능 우선순위

1. 대중 평가, 향후 성장곡선을 ai 매칭 요소에 포함해보기
    - 대중 평가
        - 대중 인식 키워드
        - 커뮤니티/팬 반응 텍스트
        - 반복되는 긍·부정 키워드
        - 연기력 관련 평가
    - 배우 성장 곡선
        - 유사 역할 반복 여부
        - 이미지 확장 중인지
        - 장르 변화 시도 경험
        - 이미지 소비율(최근 n년 내 출연작 개수)
        - 출연작 개수 대비 흥행도
2. AI가 추천한 후보 배우 리스트에 대한 팬 투표 기능 
- 후보 배우 리스트가 n 명 나오면, 팬들이 해당 리스트를 보고 투표를 할 수 있는 기능
1. 1 번 기능 과 연계하여 하나의 팬 플랫폼화 하기
- ex. 우리 배우 이번에 후보 배우 리스트로 올라왔대! 투표해서 우리 배우가 캐스팅 되게 도와주자
